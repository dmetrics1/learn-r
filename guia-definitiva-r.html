¿Qué es R y para qué sirve?

R es un lenguaje de programación y un entorno libre y gratuito, diseñado específicamente para el análisis estadístico, la ciencia de datos y la visualización.

ConceptoEcosistemaCasos de uso

Concepto

R nació en 1993 de la mano de Ross Ihaka y Robert Gentleman en la Universidad de Auckland, como una implementación libre del lenguaje S. Hoy es mantenido por el R Core Team y una enorme comunidad mundial. La versión estable más reciente es R 4.6.0 ("Because it was There", abril 2026).

A diferencia de un lenguaje de propósito general, R está pensado desde su raíz para trabajar con datos: vectores, tablas, modelos estadísticos y gráficos son ciudadanos de primera clase del lenguaje.

Casos de uso

  • Análisis estadístico y econometría (regresiones, pruebas de hipótesis, modelos mixtos).
  • Ciencia de datos y Machine Learning (clasificación, clustering, predicción).
  • Visualización de alta calidad con ggplot2, lista para publicaciones científicas.
  • Reportes reproducibles y dashboards con Quarto, R Markdown y Shiny.
  • Bioinformática, finanzas, salud pública, investigación académica y BI.

¿R o Python?

Ambos son excelentes. R brilla en estadística, visualización exploratoria y comunicación de resultados; Python destaca en producción de software, deep learning y tareas de propósito general. No es una guerra: muchos profesionales usan los dos. Si tu foco es análisis de datos y estadística, R es una de las mejores inversiones posibles.

Idea clave: en R casi todo es un vector y casi todo se hace con funciones. Interiorizar esas dos ideas hace que el resto del lenguaje encaje con naturalidad.

Instalación de R y RStudio

Necesitas dos cosas: R (el motor del lenguaje) y RStudio (el entorno donde escribirás cómodamente). Son gratuitos.

Paso 1 — Instala R (CRAN)

Descarga R desde cran.r-project.org. CRAN es el repositorio oficial. Elige tu sistema operativo y descarga la versión base más reciente (R 4.6.x).

  • Windows: Download R for Windows → base → Download R 4.6.x for Windows.
  • macOS: descarga el .pkg correspondiente a tu chip (Apple Silicon o Intel).
  • Linux: usa el gestor de paquetes (r-base) o las instrucciones de CRAN para tu distribución.

Paso 2 — Instala RStudio (Posit)

Descarga RStudio Desktop (gratis) desde posit.co. RStudio detecta automáticamente la instalación de R.

Orden correcto: instala primero R y después RStudio. RStudio es solo la interfaz; no incluye el lenguaje.

Paso 3 — Verifica la instalación

R · Consola
R.version.string   # muestra la versión instalada
sessionInfo()      # información completa de la sesión
[1] "R version 4.6.0 (2026-04-24)"

Alternativa en la nube

Si no puedes instalar nada, usa Posit Cloud: RStudio funcionando en el navegador, sin instalación. Ideal para empezar de inmediato o trabajar desde varios equipos.

Tipos de datos

Todo valor en R pertenece a un tipo. Conocerlos evita errores sutiles en cálculos y comparaciones.

Concepto

TipoEjemploDescripción
numeric / double3.14Números reales (el tipo por defecto de los números).
integer5LEnteros. La L fuerza el tipo entero.
character"texto"Cadenas de texto, entre comillas.
logicalTRUE / FALSEValores booleanos.
complex2+3iNúmeros complejos (poco frecuente).

Ejemplo: inspeccionar tipos

R
class(3.14)      # "numeric"
class(5L)        # "integer"
class("hola")    # "character"
class(TRUE)      # "logical"

# Conversión explícita (coerción)
as.integer("42")     # 42
as.numeric("3.5")    # 3.5
as.character(100)    # "100"
as.logical(0)        # FALSE
[1] "numeric" [1] "integer" [1] "character" [1] "logical"

Valores especiales

  • NA — dato faltante (Not Available).
  • NULL — ausencia de objeto (longitud cero).
  • NaN — "no es un número" (ej. 0/0).
  • Inf / -Inf — infinito (ej. 1/0).

Error frecuente: comparar con NA usando ==. x == NA siempre devuelve NA, no TRUE/FALSE. Usa siempre is.na(x).

Coerción silenciosa: si mezclas tipos, R "promueve" todo al más general. c(1, "a", TRUE) se convierte en texto. Revísalo con class() si algo no cuadra.

Ejercicio propuesto

Predice el tipo de c(1L, 2.5), de c(TRUE, 3) y de c(1, "dos"). Verifícalo con class().

Ver solución

c(1L, 2.5) → numeric (el entero se promueve a double). c(TRUE, 3) → numeric (TRUE vale 1). c(1, "dos") → character. La jerarquía de coerción es: logical → integer → double → character.

Variables y objetos

Una variable es un nombre que apunta a un valor. En R todo lo que creas (un número, una tabla, un modelo) es un objeto.

Asignación

R
edad <- 30          # operador preferido en R
nombre = "Ana"      # válido, pero menos idiomático
3.1416 -> pi_aprox  # asignación a la derecha (rara)

edad
nombre
[1] 30 [1] "Ana"

¿<- o =? La convención en R es usar <- para asignar variables y reservar = para argumentos dentro de funciones. Atajo: Alt+-.

Reglas para nombrar

  • Empieza por letra (o punto). Puede contener letras, números, . y _.
  • R distingue mayúsculas: Edad ≠ edad.
  • Evita nombres de funciones existentes (mean, data, c).

Buenas prácticas: usa snake_case (precio_total), nombres descriptivos y en un solo idioma. La guía de estilo del tidyverse es el estándar de facto.

Inspeccionar el entorno

R
ls()              # lista los objetos creados
rm(edad)          # elimina un objeto
rm(list = ls())   # elimina TODO el entorno (úsalo con cuidado)
str(nombre)       # estructura/tipo de un objeto

Ver solución
R

Operadores

Los operadores son símbolos que realizan operaciones aritméticas, de comparación y lógicas.

R
7 + 3     # 10  suma
7 - 3     # 4   resta
7 * 3     # 21  multiplicación
7 / 3     # 2.333...  división
7 %% 3    # 1   módulo (resto)
7 %/% 3   # 2   división entera
2 ^ 3     # 8   potencia
R
5 == 5    # TRUE   igualdad
5 != 3    # TRUE   distinto
5 > 3     # TRUE   mayor que
5 <= 2    # FALSE  menor o igual
"a" < "b" # TRUE   orden alfabético
R
TRUE & FALSE    # FALSE  Y (vectorizado)
TRUE | FALSE    # TRUE   O (vectorizado)
!TRUE           # FALSE  negación
# && y || evalúan solo el primer elemento (para if)
(5 > 3) && (2 < 4)   # TRUE

Una vs. dos: & y | operan elemento a elemento sobre vectores; && y || devuelven un único valor y se usan en condicionales if. Desde R 4.3 usar && con vectores de longitud >1 da error.

Error clásico: usar = en lugar de == dentro de una comparación. x = 5 asigna; x == 5 compara.

Ejercicio propuesto

¿Es 2025 bisiesto? Un año es bisiesto si es divisible por 4 y (no por 100 o sí por 400). Escríbelo en una expresión lógica.

Ver solución
R
anio <- 2025
(anio %% 4 == 0) & (anio %% 100 != 0 | anio %% 400 == 0)
# FALSE  (2025 no es bisiesto)

Vectores

El vector es la estructura más fundamental de R: una secuencia ordenada de valores del mismo tipo. Dominarlo es dominar R.

Crear vectores

R
v <- c(10, 20, 30, 40)     # c() = combine
1:5                         # 1 2 3 4 5 (secuencia)
seq(0, 1, by = 0.25)        # 0.00 0.25 0.50 0.75 1.00
rep("a", times = 3)         # "a" "a" "a"
length(v)                   # 4
[1] 10 20 30 40

Indexación (¡empieza en 1!)

R
v[1]            # 10  primer elemento
v[c(1, 3)]      # 10 30
v[-1]           # 20 30 40  (todo menos el primero)
v[v > 20]       # 30 40  indexación lógica
v[2] <- 99      # modificar un elemento

Error muy común: en R los índices empiezan en 1, no en 0 (a diferencia de Python). v[0] devuelve un vector vacío.

Vectorización: la superpotencia de R

R
precios <- c(100, 200, 300)
precios * 1.19          # aplica IVA a todos: 119 238 357
sqrt(precios)           # raíz de cada uno
sum(precios)            # 600
mean(precios)           # 200
precios + c(10, 20, 30) # suma elemento a elemento

Reciclaje: si los vectores tienen distinta longitud, R "recicla" el más corto. c(1,2,3,4) + c(10,20) → 11 22 13 24. Útil, pero cuidado: si la longitud no es múltiplo, R avisa.

Matrices

Una matriz es un vector con dos dimensiones: filas y columnas, todas del mismo tipo. Es la base del álgebra lineal en R.

Crear e indexar

R
m <- matrix(1:6, nrow = 2, ncol = 3)
m
m[1, ]      # primera fila
m[, 2]      # segunda columna
m[2, 3]     # elemento fila 2, columna 3
dim(m)      # 2 3
t(m)        # transpuesta
[,1] [,2] [,3] [1,] 1 3 5 [2,] 2 4 6

Por defecto R llena la matriz por columnas. Usa byrow = TRUE para llenar por filas.

Operaciones

R
A <- matrix(c(1,2,3,4), 2, 2)
A * 2          # multiplica cada elemento
A %*% A        # multiplicación MATRICIAL (¡no es *!)
rowSums(A); colSums(A)
rowMeans(A); colMeans(A)
solve(A)       # matriz inversa

Cuidado: A * B multiplica elemento a elemento; A %*% B es el producto matricial. Confundirlos es un error frecuente en álgebra lineal.

Ejercicio propuesto

Crea una matriz 3×3 con los números 1 al 9 por filas y obtén la suma de cada columna.

Ver solución
R
m <- matrix(1:9, nrow = 3, byrow = TRUE)
colSums(m)   # 12 15 18

Factores

Los factores representan variables categóricas: datos cualitativos con un número finito de categorías (niveles), como sexo, ciudad o nivel educativo.

Crear factores

R
tallas <- factor(c("M", "S", "L", "M", "S"))
levels(tallas)        # "L" "M" "S"  (orden alfabético)
table(tallas)         # frecuencia de cada nivel

# Factor ORDENADO (las categorías tienen jerarquía)
nivel <- factor(c("bajo","alto","medio"),
                levels = c("bajo","medio","alto"),
                ordered = TRUE)
nivel[1] < nivel[2]   # TRUE  (bajo < alto)

Internamente un factor se guarda como enteros con una "etiqueta" por nivel. Por eso son eficientes y por eso los modelos estadísticos los entienden como categorías.

Trampa peligrosa: convertir un factor de números a numérico. as.numeric(factor(c("10","20"))) devuelve 1 2 (los códigos internos), no 10 y 20. La forma correcta: as.numeric(as.character(f)).

Ver solución

              levels = c("malo","regular","bueno"), ordered = TRUE)

Listas

La lista es la estructura más flexible: puede contener elementos de distinto tipo y longitud, incluso otras listas. Los modelos y muchos resultados de R son listas.

Crear y acceder

R
persona <- list(
  nombre = "Luis",
  edad   = 28,
  hobbies = c("R", "ajedrez", "cine")
)
persona$nombre        # "Luis"  (acceso por nombre)
persona[["edad"]]     # 28      (doble corchete = el elemento)
persona["edad"]       # una sublista que contiene edad
persona$hobbies[2]    # "ajedrez"

Un corchete vs. dos: lista[1] devuelve una lista con el primer elemento; lista[[1]] devuelve el contenido del primer elemento. Es la confusión #1 con listas.

Por qué importan

Cuando ajustas un modelo, el resultado es una lista. Saber navegarla te da acceso a todo:

R
modelo <- lm(dist ~ speed, data = cars)
names(modelo)            # componentes del modelo
modelo$coefficients      # coeficientes estimados
str(modelo, max.level = 1)

Data Frames

El data frame es la estructura estrella del análisis de datos: una tabla donde cada columna es una variable (un vector) y cada fila una observación. Es el equivalente a una hoja de cálculo.

Crear y explorar

R
df <- data.frame(
  nombre = c("Ana", "Beto", "Caro"),
  edad   = c(25, 31, 28),
  ciudad = c("Cali", "Bogotá", "Medellín")
)
head(df)        # primeras filas
str(df)         # estructura: tipos y dimensiones
summary(df)     # resumen estadístico por columna
nrow(df); ncol(df); dim(df)
names(df)       # nombres de columnas

Seleccionar y filtrar (R base)

R
df$edad                 # una columna como vector
df[, "nombre"]          # idem
df[1, ]                 # primera fila completa
df[df$edad > 27, ]      # filas que cumplen una condición
df[df$edad > 27, "nombre"]   # combina filtro y columna
df$mayor <- df$edad >= 30    # crear columna nueva

R trae datasets listos para practicar: mtcars, iris, airquality. Escribe data() para ver la lista. Usaremos varios en esta guía.

Más adelante veremos el tibble (del tidyverse), una versión moderna del data frame con impresión más limpia y comportamiento más predecible.

El 90% de los proyectos empieza leyendo datos de un archivo. R lee prácticamente cualquier formato.

R

datos <- read.csv("datos.csv")

# readr (tidyverse) — más rápido y robusto
library(readr)
datos <- read_csv("datos.csv")          # separador por coma
datos <- read_csv2("datos.csv")         # separador ; y coma decimal (Europa/LatAm)

library(readxl)
datos <- read_excel("archivo.xlsx", sheet = "Hoja1")
excel_sheets("archivo.xlsx")   # ver nombres de hojas
R
library(haven)   # SPSS, Stata, SAS
read_sav("datos.sav")
read_dta("datos.dta")

library(jsonlite)
fromJSON("datos.json")

readRDS("objeto.rds")   # formato nativo de R
R
write_csv(datos, "salida.csv")
library(writexl); write_xlsx(datos, "salida.xlsx")
saveRDS(modelo, "modelo.rds")   # guarda CUALQUIER objeto de R

Funciones básicas

Una función toma argumentos y devuelve un resultado. R incluye cientos listas para usar; conocer las más comunes acelera todo tu trabajo.

Funciones que usarás a diario

CategoríaFunciones
Resumen numéricosum, mean, median, sd, var, min, max, range, quantile
Conteo / ordenlength, sort, order, rev, unique, table
Redondeoround, ceiling, floor, abs
Textopaste, paste0, nchar, toupper, substr
Lógicasany, all, which, ifelse, is.na
R
x <- c(4, 8, 15, 16, 23, 42)
mean(x); median(x); sd(x)
round(mean(x), 2)              # 18  redondeo a 2 decimales
which(x > 15)                  # posiciones: 4 5 6
ifelse(x > 15, "alto", "bajo") # vectorizado
paste("Valor:", x)             # une texto y números

Argumentos y NA

R
v <- c(10, NA, 30)
mean(v)                # NA  ← un solo NA "contamina" el resultado
mean(v, na.rm = TRUE)  # 20  ← na.rm elimina los NA antes de calcular

Error frecuentísimo: obtener NA al calcular promedios o sumas. Casi siempre es porque hay datos faltantes: añade na.rm = TRUE.

Los argumentos pueden ir por nombre o por posición. round(3.14159, 2) = round(x = 3.14159, digits = 2). Nombrarlos hace el código más legible.

Funciones personalizadas

Crear tus propias funciones es el paso que separa a quien usa R de quien programa en R. Encapsulan lógica reutilizable y reducen errores.

Anatomía de una función

R
convertir_temp <- function(celsius, escala = "F") {
  if (escala == "F") {
    resultado <- celsius * 9/5 + 32
  } else {
    resultado <- celsius + 273.15   # Kelvin
  }
  return(resultado)   # explícito (recomendado para claridad)
}

convertir_temp(25)            # 77   (Fahrenheit por defecto)
convertir_temp(25, "K")       # 298.15

R devuelve automáticamente el valor de la última expresión, así que return() es opcional. Aun así, muchos lo usan para hacer explícito el resultado.

Argumentos por defecto y ...

R
resumen <- function(x, ...) {
  c(media = mean(x, ...), de = sd(x, ...))
}
resumen(c(1, 2, NA, 4), na.rm = TRUE)   # los ... pasan na.rm a mean/sd

Sintaxis corta (R 4.1+)

R
cuadrado <- \(x) x^2     # equivale a function(x) x^2
cuadrado(6)              # 36

Buenas prácticas: una función = una responsabilidad. Nombres en verbo (calcular_iva), argumentos en orden lógico (datos primero), y evita modificar variables globales desde dentro.

Manejo de errores

Código robusto anticipa lo que puede salir mal. R distingue tres condiciones: errores (detienen), warnings (avisan) y messages (informan).

tryCatch

R
leer_seguro <- function(ruta) {
  tryCatch(
    expr    = read.csv(ruta),
    error   = function(e) {
      message("No se pudo leer: ", conditionMessage(e))
      NULL
    },
    warning = function(w) { message("Aviso: ", conditionMessage(w)); NULL },
    finally = message("Intento de lectura finalizado.")
  )
}

Validar argumentos y avisar

R
raiz <- function(x) {
  if (!is.numeric(x))  stop("x debe ser numérico")
  if (any(x < 0))      warning("Hay negativos: producen NaN")
  sqrt(x)
}

# stopifnot: validación rápida
media_pos <- function(x) {
  stopifnot(is.numeric(x), length(x) > 0)
  mean(x)
}

El paquete rlang ofrece abort(), warn() e inform() con mensajes más ricos. purrr::safely() y possibly() envuelven funciones para que nunca detengan un pipeline.

Buena práctica: falla con mensajes accionables. "x debe ser numérico, recibí character" es mucho mejor que "Error en sqrt".

El pipe: el corazón del flujo

El operador pipe encadena operaciones: el resultado de la izquierda pasa como primer argumento de la derecha. Hace el código legible de izquierda a derecha, como una receta.

R
library(tidyverse)

# Sin pipe (anidado, difícil de leer)
arrange(summarise(group_by(mtcars, cyl), mpg_med = mean(mpg)), mpg_med)

# Con pipe nativo |> (R 4.1+) — se lee de arriba a abajo
mtcars |>
  group_by(cyl) |>
  summarise(mpg_med = mean(mpg)) |>
  arrange(mpg_med)

Dos pipes: |> es el pipe nativo de R (recomendado hoy); %>% es el de magrittr, aún muy usado. Atajo en RStudio: Ctrl/Cmd+Shift+M.

Concepto clave: datos ordenados (tidy data) = cada variable en una columna, cada observación en una fila, cada valor en una celda. Casi todo el tidyverse asume este formato.

Programación funcional

R es, en el fondo, un lenguaje funcional: las funciones son objetos de primera clase. El paquete purrr lleva este paradigma a su máxima expresión.

map: iterar sin bucles

map_dbl(1:3, \(x) x^2)         # devuelve vector double: 1 4 9

Optimización de código

Antes de optimizar, mide. "La optimización prematura es la raíz de todos los males" (Knuth). Identifica el cuello de botella real y solo entonces actúa.

Medir (profiling)

R
library(bench)
mark(
  bucle    = { s <- 0; for (i in 1:1e5) s <- s + i; s },
  vectoriz = sum(1:1e5)
)               # compara tiempo y memoria

Estrategias de mayor a menor impacto

  1. Vectoriza: reemplaza bucles por operaciones vectorizadas y funciones map/apply.
  2. Pre-asigna la memoria; no hagas crecer vectores en bucles.
  3. Usa estructuras eficientes: data.table para datos grandes.
  4. Paraleliza: furrr::future_map() usa varios núcleos.
  5. Reescribe lo crítico en C++ con Rcpp (último recurso).

El 80% de las mejoras viene del 20% del código. No optimices funciones que se ejecutan una vez; concéntrate en lo que corre millones de veces.

data.table

data.table es la herramienta de referencia para datos grandes en R: extremadamente rápida y eficiente en memoria, con una sintaxis concisa.

La sintaxis dt[i, j, by]

Se lee como: "toma las filas i, calcula j, agrupando por by".

R · data.table
library(data.table)
dt <- as.data.table(mtcars)

dt[mpg > 20]                              # filtrar filas
dt[, .(mpg_medio = mean(mpg)), by = cyl]  # resumir por grupo
dt[, peso_kg := wt * 453.6]               # crear columna POR REFERENCIA (:=)
dt[order(-mpg)]                           # ordenar

# Encadenado
dt[mpg > 15][, .(hp_max = max(hp)), by = gear]

El operador := modifica por referencia (sin copiar la tabla), lo que ahorra muchísima memoria con datasets de millones de filas. Es la clave de su velocidad.

¿dplyr o data.table? dplyr es más legible y suficiente para la mayoría de casos; data.table gana cuando el rendimiento es crítico (decenas de millones de filas). El paquete dtplyr te deja escribir dplyr y ejecutar con el motor de data.table.

Machine Learning

R tiene un ecosistema de ML maduro. El framework moderno es tidymodels, que unifica el flujo de trabajo con una gramática consistente.

El flujo de tidymodels

Dividir → preprocesar (recipe) → especificar modelo → entrenar (workflow) → evaluar → ajustar (tuning).

R · tidymodels
library(tidymodels)

# 1. Dividir datos (entrenamiento / prueba)
set.seed(123)
split  <- initial_split(iris, prop = 0.8, strata = Species)
train  <- training(split); test <- testing(split)

# 2. Receta de preprocesamiento
receta <- recipe(Species ~ ., data = train) |>
  step_normalize(all_numeric_predictors())

# 3. Especificar modelo
modelo <- rand_forest(trees = 500) |>
  set_engine("ranger") |>
  set_mode("classification")

# 4. Workflow + entrenamiento
wf <- workflow() |> add_recipe(receta) |> add_model(modelo)
fit <- wf |> fit(data = train)

# 5. Evaluar
pred <- predict(fit, test) |> bind_cols(test)
accuracy(pred, truth = Species, estimate = .pred_class)

Sobreajuste (overfitting): nunca evalúes en los mismos datos con que entrenaste. Usa validación cruzada (vfold_cv()) y reserva un conjunto de prueba que el modelo no vea hasta el final.

Buenas prácticas y arquitectura de proyectos

Un proyecto profesional es legible, reproducible y mantenible. La estructura importa tanto como el código.

Estructura recomendada de un proyecto

proyecto/
mi_proyecto/
├── mi_proyecto.Rproj    # proyecto de RStudio
├── README.md            # qué hace y cómo correrlo
├── renv.lock            # versiones de paquetes
├── data/
│   ├── raw/             # datos originales (nunca se modifican)
│   └── processed/       # datos limpios generados
├── R/                   # funciones reutilizables
├── scripts/             # 01_limpieza.R, 02_analisis.R...
├── output/              # gráficos, tablas, modelos
└── reports/             # informes Quarto

Principios clave

  • Reproducibilidad: cualquiera debe poder clonar y ejecutar. Usa renv y rutas relativas (here).
  • Estilo consistente: sigue la guía del tidyverse; automatiza con styler y lintr.
  • No te repitas (DRY): si copias código tres veces, hazlo una función.
  • Separa datos crudos de procesados: los crudos son sagrados; nunca los sobrescribas.
  • Documenta y testea lo importante.

Para flujos complejos y reproducibles, el paquete targets orquesta tu pipeline: solo recalcula lo que cambió, como un Makefile para R.

Integra todo: desde los datos crudos hasta un modelo desplegado. Este es el proyecto que pondrás en tu portafolio.

El problema

Predecir qué clientes abandonarán un servicio (churn) y desplegar el modelo como API + dashboard. Es un caso clásico y muy valorado en empresas.

Las 7 etapas

  1. Ingesta: cargar datos (CSV/SQL) con un proyecto reproducible.
  2. Limpieza y EDA: tidyverse + visualización para entender el problema.
  3. Feature engineering: crear variables predictivas con recipes.
  4. Modelado: entrenar y comparar modelos con tidymodels + validación cruzada.
  5. Evaluación: métricas adecuadas (recall, AUC) e interpretabilidad (vip, SHAP).
  6. Despliegue: exponer el modelo con plumber + un dashboard Shiny.
  7. Documentación: reporte Quarto y README, todo versionado en GitHub.
R · esqueleto del pipeline
library(tidymodels)

# Datos (ejemplo: telco churn)
datos <- read_csv("data/raw/churn.csv") |>
  mutate(churn = factor(churn))

set.seed(42)
split <- initial_split(datos, prop = 0.8, strata = churn)
folds <- vfold_cv(training(split), v = 5, strata = churn)

receta <- recipe(churn ~ ., data = training(split)) |>
  step_rm(id_cliente) |>
  step_impute_median(all_numeric_predictors()) |>
  step_dummy(all_nominal_predictors()) |>
  step_normalize(all_numeric_predictors())

modelo <- boost_tree(trees = 500, learn_rate = 0.05) |>
  set_engine("xgboost") |> set_mode("classification")

wf  <- workflow() |> add_recipe(receta) |> add_model(modelo)
res <- fit_resamples(wf, folds, metrics = metric_set(roc_auc, recall))
collect_metrics(res)

final <- last_fit(wf, split)
collect_metrics(final)         # desempeño en el conjunto de prueba
